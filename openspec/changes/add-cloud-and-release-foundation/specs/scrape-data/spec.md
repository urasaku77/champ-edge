# scrape-data 変更デルタ

## MODIFIED Requirements

### Requirement: スクレイピングデータのローカルキャッシュ

システムはスクレイピング由来データをローカルキャッシュとして保持し、機能はそれを読むだけにし**なければならない（MUST）**。対象は全体使用率ランキング・シーズン定義・構築記事・HOME使用率（旧 champ-edge の
stats/ranking.txt・recog/season.json・構築記事DB・stats/home_*.csv に相当）。読み込みは**配信レイヤ `RefData`
経由**で、**取得済みキャッシュがあればそれ、無ければアプリ同梱の seed** を使う（`assets/data/scrape/*` と
`assets/data/home/*`）。これによりオフラインや配信未整備でも常に動作する。

データの**取得（更新）は機能本体と切り分ける**：`RefData` が無料の静的配信（クラウド）から取得して
ローカルキャッシュを更新する **stale-while-revalidate** 方式とし、更新分は次回読み込みから反映する。
クライアント側スクレイピングは行わない（サーバー集約データを取得するだけ）。取得の自動実行は配信元が
整備されるまで無効とし、当面は**設定画面の手動更新**でのみ取得する（既定は同梱データ）。

#### Scenario: seed が無くても機能が壊れない

- **WHEN** スクレイピングデータが空または未整備のまま機能を使う
- **THEN** 例外を出さず、ランキング依存表示は無効化・シーズンは非表示・構築記事一致は0件として動作する

#### Scenario: キャッシュ優先・同梱フォールバック

- **WHEN** クラウド取得済みキャッシュがある状態で参照データを読む
- **THEN** キャッシュを使う（無ければ同梱 seed にフォールバックし、オフラインでも動作する）

#### Scenario: 手動更新

- **WHEN** 設定画面で「参照データを更新」を実行する
- **THEN** クラウドから最新を取得してキャッシュを更新し、次回読み込みから反映する（失敗時は同梱のまま）
