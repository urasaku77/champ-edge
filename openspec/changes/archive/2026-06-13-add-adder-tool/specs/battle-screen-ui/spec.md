# 加算ツール・技詳細ポップアップ

## ADDED Requirements

### Requirement: 加算ツール

システムは複数技の合算ダメージを確認できるようにし**なければならない（MUST）**。技行のダメージ/割合部を
長押しすると加算ツールのポップアップを開く（旧 champ-edge の `MultiWazaDamageWindow`＝ダメージ表示
右クリック相当）：

- 攻撃側の設定済み技がボタンで並び、押すたびにその技のダメージが累積リストへ追加される
  （同じ技を複数回押すと複数回分加算）。
- 回復ボタン（オボンのみ＝最大HP×1/4・たべのこし＝×1/16・混乱実＝×1/3・やどりぎのタネ＝×1/8）を
  押すと回復量が減算行として追加される。
- 各行は 技名/回復名・ダメージ・割合（技は単発の確定数付き）を表示し、**行をタップするとその行だけ
  削除**して再計算される。「クリア」ボタンで全行リセット。
- 合計欄にネットダメージ実数（min−max）・割合・**KO 判定**を表示する。KO 判定は各技の 16 値ダメージ
  分布（定数ダメージ加算済み）の直積を DP で集計し、しきい値＝実効HP（ステルスロック分を差し引き、
  最低1）＋回復合計 とし、全組み合わせで KO なら「確定KO」、一部なら「乱数KO (確率%)」とする。
- やけど・急所・定数ダメージ等の設定は各技のダメージに反映済みの値を用いる。

#### Scenario: 複数技の合算と KO 判定

- **WHEN** 加算ツールで技Aを2回押し、技Bを1回押す
- **THEN** 3行が追加され、合計欄に合算ダメージ・割合と 確定KO/乱数KO(確率%) が表示される

#### Scenario: 回復の減算と行削除

- **WHEN** たべのこしボタンを押し、その後いずれかの行をタップする
- **THEN** 回復分が合計から減算され、タップした行だけが削除されて再計算される

### Requirement: 技詳細の表示

システムは技の詳細を確認できるようにし**なければならない（MUST）**。技行の技名を長押しすると、
`waza_data` の タイプ・分類・威力・命中・PP・接触/非接触・まもる可否・対象 と説明文（description）を
ダイアログで表示する（旧 champ-edge の技右クリック相当）。特性・持ち物の長押し効果詳細と同じ流儀。

#### Scenario: 技詳細を見る

- **WHEN** 設定済みの技の技名を長押しする
- **THEN** その技のタイプ/分類/威力/命中/PP と説明文がダイアログ表示される

## MODIFIED Requirements

### Requirement: 技ごとのダメージ可視化

システムは各技のダメージを可視化し**なければならない（MUST）**。各技についてダメージ範囲（min〜max）、
防御側 HP に対する割合（min–max%）、確定数を表示すること。確定数は旧 champ-edge の `ko_text` 準拠で、
n 発（1→4）で 16/16 倒せれば「確定 n 発」、それ未満は「乱数 n 発（k/16 の分数付き）」と表す。
技行は 1 行で「技名（左・固定幅、威力表示は省略）／ダメージ範囲（中央）／割合・確定数（右寄せ）」を並べ、
確定数の桁変化でも行がズレないこと。バーは旧 champ-edge
の HP バー準拠で**残り HP**を表し（明色＝確定で残る、暗色＝min〜max の不確定帯、灰＝確定で失う）、
色は最大ダメージ割合で 赤（≥80%）／黄（≥50%）／緑（<50%）とする。技は 5 枠（自分は空白可、相手は
5 枠目に暫定技）を横画面でスクロール無しに収める。
**長押し操作の領域分割**：技名の長押しで技詳細（「技詳細の表示」）、ダメージ/割合部の長押しで
加算ツール（「加算ツール」）を開く。タップは従来通り技変更。

#### Scenario: ダメージ割合の可視化

- **WHEN** ある技を見る
- **THEN** その技のダメージ範囲・対 HP 割合・確定数が表示され、残り HP バーが割合に応じた色で示される

#### Scenario: 長押しの領域分割

- **WHEN** 技名を長押しする／ダメージ部を長押しする
- **THEN** 技詳細ダイアログ／加算ツールがそれぞれ開く
