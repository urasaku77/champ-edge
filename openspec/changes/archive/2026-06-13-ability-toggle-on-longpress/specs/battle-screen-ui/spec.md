# 特性の有効/無効を長押しで即時切替

## MODIFIED Requirements

### Requirement: 特性・持ち物の編集

選択ポケモンの特性・持ち物をタップで変更できなければならない（MUST）。特性はそのポケモンの
候補（DB の ability1/2/3、未接続時はサンプル）から、持ち物は**文字入力検索**（DB の item_data）で
選ぶ。item_data は表記が混在（カタカナ/ひらがな）のため、検索は**生入力・カタカナ・ひらがなのいずれかで
部分一致**させる（いのちのたま／こだわりスカーフ等いずれも引ける）。**特性・持ち物の選択時は、その相手
ポケモンの HOME 使用率を各候補の横に表示し、使用率の高い順に並べ替える**。変更するとダメージが
再計算されること。チップは選択値のみ表示し、持ち物が「なし」のときは「もちものなし」と表示する。
さらに、**特性・持ち物のチップをダブルタップすると、その効果詳細（`ability_data.effect` /
`item_data.effect`）をダイアログ表示する**（旧 champ-edge の右クリック効果詳細に相当）。

**特性値の切替**：旧 champ-edge の `ABILITY_VALUES` を移植し、条件付き特性は値を切り替えられること。
有効/無効型（しんりょく・もうか・げきりゅう・スナイパー・むしのしらせ・ねつぼうそう・どくぼうそう・
プラス・マイナス＝デフォルト無効／へんげんじざい・リベロ・マルチスケイル・ファントムガード・
ふしぎなうろこ・アナライズ・かたやぶり・スロースタート・よわき・テラスシェル＝デフォルト有効／
ふかしのこぶし・かんつうドリル＝デフォルト無効・有効でダメージ×1/4）と、多値型（そうだいしょう
1.0/1.1/1.2・とうそうしん 1.0/1.25/0.75・こだいかっせい/クォークチャージ なし/A/B/C/D/S）がある。

- **特性チップの長押しで有効/無効（値）を即時切替**する（ダイアログを開かず、結果をスナックバーで表示）。
  値を持つ特性はリストを循環（有効/無効はトグル、多値は順送り）、登場時ランク特性（いかく等）は
  登場時ランク効果の有効/無効をトグル、切替値のない特性は「切替なし」を示す。
- ダブルタップの効果詳細ダイアログ内のチップ/スイッチでも同じ切替ができる。いずれも即座にダメージへ反映する。
- **特性を変更すると、その特性のデフォルト値（テーブル先頭）へ自動リセット**する
  （原典 `set_default_ability_value` と同一）。値はパーティに永続化する。
- 値を持つ特性はチップに現在値を `[値]` で併記する。
- 値はエンジンの `abilityValue` へ渡り、条件付き特性補正（マルチスケイル半減・こだいかっせい等）が
  実際に計算へ反映されること。

#### Scenario: 特性の変更

- **WHEN** 特性をタップして候補から別の特性を選ぶ
- **THEN** 選択ポケモンの特性が更新され（特性値はデフォルトへリセット）、ダメージが再計算される

#### Scenario: 長押しで有効/無効を即時切替

- **WHEN** ふかしのこぶし（デフォルト無効）の特性チップを長押しする
- **THEN** ダイアログを開かずに値が「有効」へ切り替わり（再長押しで「無効」へ戻る）、ダメージが再計算される

#### Scenario: マルチスケイルが防御計算に乗る

- **WHEN** 防御側がマルチスケイル（デフォルト有効）を持つ
- **THEN** 被ダメージが半減して計算される（長押しで「無効」にすると通常に戻る）

#### Scenario: 持ち物の検索（混在表記）

- **WHEN** 持ち物検索で「いのち」と入力する
- **THEN** ひらがな表記の「いのちのたま」が候補に表示される

#### Scenario: 効果詳細の表示

- **WHEN** 特性チップ（または持ち物チップ）をダブルタップする
- **THEN** その特性/持ち物の効果説明がダイアログで表示される
