# 類似パーティ検索

## ADDED Requirements

### Requirement: 類似パーティ検索

システムは相手の選択パーティに一致する過去のパーティを検索できるようにし**なければならない（MUST）**
（旧 champ-edge の類似パーティ）。中央列の「類似パーティ検索」アイコンから結果ダイアログを開き、相手
パーティ6体と候補を照合して、**並びまで一致（exactOrder）／中身だけ同じ（sameSet）**を区別して表示する。
照合はメガ進化フォームの統合（form 10-19→通常）に対応し、UI 非依存に分離する。

検索対象は次の2系統:

- **対戦履歴ベース**：過去の対戦記録の相手パーティ（重複排除）から一致を探す。
- **構築記事ベース**：構築記事ストア（タイトル・URL・6体）から一致を探し、構築記事リンクを開ける。
  構築記事データはサーバー集約で投入する想定（クライアント側スクレイピングは行わない）。

#### Scenario: 並び一致と中身一致を区別する

- **WHEN** 類似パーティ検索を開く
- **THEN** 相手パーティと並びまで一致する候補は「並び一致」、構成のみ一致する候補は「中身一致」として
  区別表示される

#### Scenario: 構築記事リンクを開く

- **WHEN** 構築記事の一致結果の「構築記事」リンクを押す
- **THEN** 対応する構築記事の URL が外部ブラウザで開く
