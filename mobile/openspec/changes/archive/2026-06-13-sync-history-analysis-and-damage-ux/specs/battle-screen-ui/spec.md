# battle-screen-ui 変更デルタ

## MODIFIED Requirements

### Requirement: ステータス（努力値・ランク）の編集

システムは努力値とランクを編集できるようにし**なければならない（MUST）**。努力値は各 0〜32
スケール（champ-edge 準拠）で、**0／32 のワンクリックボタンと ±1 ステップボタン**で調整する（数値は
中央に表示）。ランクは各 −6〜+6（−／＋ステッパ）。実数値の
表示部をタップするとポップアップが開き（編集アイコンは置かない）、能力名・実数値・努力値・ランクを
整列して 1 画面に収めスクロール無しで表示すること。**努力値合計は努力値の列の下に表示**する。**ランクの
クリアは「ランククリア」ラベル＋掃き出しアイコンのボタン**で行い、全能力のランクを 0 に戻すと分かるよう
にする。加えて**メイン画面の実数値表示（ステータス表）を長押しすると全能力のランクを一括クリア**できる。
変更すると実数値とダメージが即時再計算されること。レベルは 50 固定、個体値は内部 31 固定で UI には出さない。

#### Scenario: 努力値変更で実数値が変わる

- **WHEN** ある能力の「32」ボタン（または＋ボタン）を押す
- **THEN** その能力の努力値が 32（または +1）になり、実数値とダメージが即時再計算される

#### Scenario: ランククリアでランクが 0 に戻る

- **WHEN** 「ランククリア」ボタンを押す
- **THEN** 全能力のランクが 0 に戻り、実数値とダメージが再計算される

#### Scenario: ステータス表の長押しでランク一括クリア

- **WHEN** メイン画面の実数値表示（ステータス表）を長押しする
- **THEN** 全能力のランクが 0 に戻り、実数値とダメージが再計算される

### Requirement: 定数ダメージの指定

システムは防御側の定数ダメージを指定できるようにし**なければならない（MUST）**。状態行（やけど/じゅうでん/
うちおとす/壁 の並び）に「定数」チップを置き、タップでダイアログを開く。ダイアログは**ステルスロック**
（登場時1回・いわ相性で算出）と**まきびし**（登場時1回・枚数で 1/8・1/6・1/4。タップで 0→1→2→3 を循環）の
切替と、**毎ターンのチップダメージ**（やどりぎ 1/8・砂嵐 1/16・どく 1/8・やけど 1/16・のろい 1/4・
ゴツゴツメット 1/6 等）の加算／クリアを提供する。表示ダメージ・割合には毎ターンのチップ（floor(HP×割合)）を
加算し、確定数は旧 `ko_text` 準拠で entry（ステロ＋まきびし）を引いた実効 HP に対し毎ターン constant×n を
加味して算出する。**定数が設定されているときはチップに値のみ表示**し（「定数」ラベルは省く・まきびしは枚数を
併記）、未設定時のみ「定数なし」と表示する。ダイアログの**クリアは毎ターン定数に加えてステロ・まきびしも
一括クリア**する。

#### Scenario: まきびしで確定数が変わる

- **WHEN** 定数ダイアログでまきびしを2回タップして2枚（1/6）にする
- **THEN** 登場時に floor(HP×1/6) を引いた実効 HP で確定数が再計算される

#### Scenario: 定数クリアでハザードも消える

- **WHEN** 定数ダイアログの「クリア」を押す
- **THEN** 毎ターン定数・ステルスロック・まきびしがすべて 0/OFF になる

### Requirement: HOME 使用率情報の表示（相手スカウト）

システムは HOME（pokedb.tokyo 集計）の使用率情報を表示でき**なければならない（MUST）**。中央カラムの
ハンバーガーメニューの下に HOME アイコンを置き、タップで**相手（選択中）のポケモン**の使用率情報を
**画面いっぱいのポップアップ**で表示する（旧 champ-edge の HomeFrame＝player1 専用に相当）。ポップアップは
もちもの・とくせい・せいかく・どりょくち の4カテゴリを横並びにし、各カテゴリ内は値と使用率（%）を
使用率降順で縦に並べ、行タップでその値を相手ポケモンへ適用（努力値は 0-32 反映）する。**現在その
ポケモンに設定済みの値は塗りつぶしで強調**し、同じ値を再タップすると未選択（既定値：持ち物=なし／
性格=まじめ／努力値=0／特性=第一候補）に戻す。**わざ**の使用率は HOME ポップアップではなく
**技検索の候補先頭**に割合付きで提示する（タップで技確定）。**相手ポケモンの技枠は5枠あり、選択（選出）時に
ダメージ計算用へ正規化する**：変化技（状態技）を除外して**物理・特殊のみ**を残し、空いた枠を HOME 使用率
上位（非変化・重複なし）で5枠まで補完する（5枠目＝スカウト枠）。データは旧アプリの `stats/home_*.csv`
（pokedb.tokyo のスナップショット）をアセット同梱して読む（実行時はローカル CSV を読むのみ。スクレイプ
更新は対象外）。メガなどフォーム名は基本形名へ正規化して引く。

#### Scenario: HOME 使用率の表示

- **WHEN** HOME アイコンをタップする
- **THEN** 相手の選択ポケモンの もちもの・とくせい・せいかく・どりょくち が使用率付きで縦並び表示される

#### Scenario: 相手技の5枠正規化

- **WHEN** 相手のポケモンを選出（選択）する
- **THEN** 変化技は表示から除外され、物理・特殊の技が HOME 使用率上位で5枠まで補完される

#### Scenario: 技検索での HOME 上位提示

- **WHEN** 相手の技枠をタップして技検索を開く
- **THEN** そのポケモンの HOME 使用率上位の技が候補先頭に割合付きで表示され、タップで技が確定する

## ADDED Requirements

### Requirement: スクロール挙動（収まる時は固定）

システムは各画面のスクロールを、**内容が画面に収まっているときは無効**にし**なければならない（MUST）**。
内容がはみ出したときだけスクロールできる（iOS 既定の弾み＝BouncingScrollPhysics は内容が収まっていても
動くため、アプリ全体のスクロール物理を ClampingScrollPhysics に統一する）。特にメイン（Top）画面と設定画面で、
収まっている状態では画面が動かないようにする。メイン画面の対戦パネルは技5枠＋基本情報が収まるよう余白を詰める。

#### Scenario: 収まっているときは動かない

- **WHEN** メイン画面で内容が画面内に収まっている状態でドラッグする
- **THEN** 画面はスクロール（弾み）せず固定される

#### Scenario: はみ出したときだけスクロール

- **WHEN** 内容が画面の高さを超える
- **THEN** その分だけスクロールできる
