# 対戦画面 UI 仕様

## ADDED Requirements

### Requirement: 左右対戦レイアウト

対戦画面は横画面・全画面で表示され、左に自分・右に相手のパネルを並べ**なければならない（MUST）**。
タイトルバーは表示せず、各パネルは独立してスクロールできること。各パネルは上から
「パーティ（選出番号付き）」「選択ポケモンの基本情報」「技とダメージ」を縦に並べる。

#### Scenario: 2 カラムの表示

- **WHEN** 対戦画面を横画面で開く
- **THEN** 左に自分、右に相手のパネルが並び、各パネルにパーティ・選択ポケモン情報・技が表示される

#### Scenario: タイトル非表示と全画面

- **WHEN** 対戦画面を表示する
- **THEN** アプリタイトルバーは表示されず、画面全体がダメージ計算 UI に使われる

### Requirement: パーティと選出（画像・番号表示・最大3体）

各側はパーティ 6 体を**画像**で横並び表示し**なければならない（MUST）**。選出は独立した欄を設けず、
パーティのポケモンに**選んだ順の番号（1〜3、最大 3 体）**を重ねて表示すること。パーティの
ポケモンをタップすると詳細表示の対象（選択ポケモン）になり、選出が 3 体未満なら選出に追加される。
選出が 3 体のときは通常タップでは選出を変えず、**長押しで選出から解除**する。

#### Scenario: タップで選出と詳細切替

- **WHEN** 選出が 3 体未満の状態でパーティのポケモンをタップする
- **THEN** そのポケモンが選出に番号付きで追加され、選択ポケモンの基本情報・技がそのポケモンに更新される

#### Scenario: 3 体選出済みでのタップ

- **WHEN** 既に 3 体選出済みの状態で未選出のポケモンをタップする
- **THEN** 選出は変化せず、詳細表示のみそのポケモンに切り替わる

#### Scenario: 長押しで選出解除

- **WHEN** 選出中のポケモンをパーティで長押しする
- **THEN** そのポケモンが選出から外れる

### Requirement: パーティの手動編集

システムは各パーティを手動で編集できるようにし**なければならない（MUST）**。パーティ横の編集ボタンから
編集ダイアログを開き、6 枠のいずれかをタップすると**ポケモンを文字入力検索**（`pokemon_data`）して
そのポケモン（種族値・タイプ・特性を DB から構築）に差し替えられること。編集ダイアログには**クリア
ボタン**があり、6 枠すべてを空にできる。差し替えたポケモンの技は未設定（タップで技検索）から始まる。
ポケモン候補は旧 champ-edge と同様に**メガシンカ（name LIKE 'メガ%'、メガニウム除く）と
除外リスト（ヒヒダルマ(ダルマ) 等）を外し、さらに画像があるポケモンのみ**に限定する。
直近の自分・相手パーティは自動保存し、次回起動時にそのまま復元する（旧アプリの「使用中」パーティ相当）。
名前付きの保存・読込 UI は当面設けない（必要になれば後続フェーズで追加）。

#### Scenario: 枠のポケモン差し替え

- **WHEN** パーティ編集ボタンを押し、編集ダイアログの枠をタップしてポケモンを検索・選択する
- **THEN** その枠が選んだポケモン（DB の種族値・タイプ・特性）に差し替わり、対戦画面に反映される

#### Scenario: クリア

- **WHEN** 編集ダイアログのクリアボタンを押す
- **THEN** そのパーティの 6 枠がすべて空になる

### Requirement: 選択ポケモンの基本情報表示

システムは選択中のポケモンの基本情報を表示し**なければならない（MUST）**。表示項目は画像・名前・
タイプ・特性・持ち物・性格・実数値（H/A/B/C/D/S）・努力値・ランクとする。タイプは小さめの
チップで**ポケモン名の横**に表示し（複数タイプは縦2段、単タイプは1つだけ）、**タップで変更**
できること（バトル中のタイプ変更に対応）。
実数値は種族値・個体値・努力値（0〜32 スケール）・レベル・性格
から算出すること。テラスタイプとレベルは当面表示しない（テラスは将来再導入候補）。

#### Scenario: 基本情報の表示

- **WHEN** ポケモンを選択する
- **THEN** 画像・名前・タイプ（名前の横）・特性・持ち物・性格・実数値・努力値・ランクが表示される

#### Scenario: タイプ変更

- **WHEN** タイプをタップしてタイプ選択ダイアログから別タイプを選ぶ
- **THEN** そのタイプ枠が変更され、タイプ一致補正・相性を反映してダメージが再計算される

### Requirement: メガ進化（フォーム切替）

選択ポケモンの画像をタップするとメガ進化（フォーム切替）でき**なければならない（MUST）**。旧 champ-edge の
フォームボタンと同様に、メガフォームを持つポケモン（`pokemon_data` の form 10-19）はタップで
通常→メガ→（複数あれば次メガ）→通常 と循環し、種族値・タイプ・特性・名前・画像をそのフォームに
差し替える（技・努力値・性格・持ち物・ランク等は維持）。メガフォームを持たないポケモンは変化しない。
種族値ポップアップや外部サイトリンクは設けない。

#### Scenario: タップでメガ進化

- **WHEN** メガフォームを持つポケモンの画像をタップする
- **THEN** メガフォームに切り替わり、種族値・タイプ・特性を反映してダメージが再計算される

#### Scenario: 元フォームへ戻る

- **WHEN** メガ状態のポケモンの画像をさらにタップする（最終メガの次）
- **THEN** 元のフォームに戻る

### Requirement: 技の文字入力検索による選択

技は文字入力で検索して選択できる仕組みを提供し**なければならない（MUST）**。技名の一部を
入力すると候補が絞り込まれ、選んだ技がその枠に差し替わり、ダメージが再計算されること。
技候補は `pokemon.db` の `waza_data` から取得する（DB 未接続時はサンプル技プールで代替）。

#### Scenario: 技名検索で差し替え

- **WHEN** 技枠をタップし、検索欄に技名の一部を入力する
- **THEN** 一致する技候補が表示され、選択した技がその枠に設定されダメージが再計算される

### Requirement: 特性・持ち物の編集

選択ポケモンの特性・持ち物をタップで変更できなければならない（MUST）。特性はそのポケモンの
候補（DB の ability1/2/3、未接続時はサンプル）から、持ち物は**文字入力検索**（DB の item_data、
ひらがな入力もカタカナに変換して検索）で選ぶ。**特性・持ち物の選択時は、その相手ポケモンの HOME
使用率を各候補の横に表示し、使用率の高い順に並べ替える**。変更するとダメージが再計算されること。
チップは選択値のみ表示し、持ち物が「なし」のときは「もちものなし」と表示する。

#### Scenario: 特性の変更

- **WHEN** 特性をタップして候補から別の特性を選ぶ
- **THEN** 選択ポケモンの特性が更新され、特性補正を含めてダメージが再計算される

#### Scenario: 持ち物の変更

- **WHEN** 持ち物をタップして別の持ち物を選ぶ
- **THEN** 選択ポケモンの持ち物が更新され、持ち物補正を含めてダメージが再計算される

### Requirement: 性格の表形式選択

性格は 5×5 の表形式で選択できなければならない（MUST）。行を上昇能力（↑）、列を下降能力（↓）
とし、対角線上は補正なし（まじめ等）を表す。選択すると実数値とダメージが再計算されること。

#### Scenario: 表形式での性格選択

- **WHEN** 性格をタップし、表の「↑こうげき × ↓とくこう」のセルを選ぶ
- **THEN** 性格が「いじっぱり」になり、攻撃の実数値が上がり特攻が下がってダメージが再計算される

### Requirement: ステータス（努力値・ランク）の編集

システムは努力値とランクを編集できるようにし**なければならない（MUST）**。努力値は各 0〜32
スケール（champ-edge 準拠、スライダーで調整）、ランクは各 −6〜+6（−／＋ステッパ）。実数値の
表示部をタップするとポップアップが開き（編集アイコンは置かない）、能力名・実数値・努力値・ランクを
整列して 1 画面に収めスクロール無しで表示すること。変更すると実数値とダメージが即時再計算される
こと。レベルは 50 固定、個体値は内部 31 固定で UI には出さない。

#### Scenario: 努力値変更で実数値が変わる

- **WHEN** 攻撃の努力値を変更する
- **THEN** 攻撃の実数値が再計算され、その値を用いてダメージが再計算される

#### Scenario: ランク変更でダメージが変わる

- **WHEN** 攻撃のランクを +2 にする
- **THEN** ランク補正後の攻撃実数値でダメージが再計算される

### Requirement: 状態異常・じゅうでん・急所の指定

システムは「やけど」「じゅうでん」「急所」を指定できるようにし**なければならない（MUST）**。
旧 champ-edge と同様にチェックボックスで指定する。状態異常はやけどのみを対象とする。
急所はポケモン名と同じ行の右端に置く。これらは計算に反映されること。なお「うちおとす」は
防御側の飛行タイプを外すことで代替できるため UI からは省く（エンジンは smackdown を保持）。

#### Scenario: やけどで物理ダメージが下がる

- **WHEN** 攻撃側ポケモンの「やけど」を ON にする
- **THEN** 物理技のダメージが約半分に再計算される（からげんき等の例外を除く）

#### Scenario: じゅうでんででんき技が強化される

- **WHEN** 攻撃側の「じゅうでん」を ON にする
- **THEN** でんきタイプの技のダメージが 2 倍に再計算される

#### Scenario: 急所

- **WHEN** 「急所」を ON にする
- **THEN** 急所補正（1.5 倍・有利ランク無視）を反映してダメージが再計算される

### Requirement: 天候・フィールド・壁の指定

システムは天候・フィールド・壁を指定できるようにし**なければならない（MUST）**。天候（なし/晴れ/雨/
砂嵐/雪）とフィールド（なし/エレキ/サイコ/グラス/ミスト）は**中央列**（メニュー/HOME アイコンを縦に
並べたその下のドロップダウン）から、壁（壁なし/物理壁＝リフレクター/特殊壁＝ひかりのかべ/両壁＝
オーロラベール）は選択ポケモンから指定でき、いずれも計算に反映されること。テラスタイプ変更は当面
対象外（将来再導入候補）。

#### Scenario: 天候変更でダメージが変わる

- **WHEN** 中央列の天候を「雨」にする
- **THEN** みずタイプ技が強化され、ほのおタイプ技が弱化されてダメージが再計算される

#### Scenario: 壁でダメージが半減する

- **WHEN** 防御側に「リフレクター」を設定する
- **THEN** 物理技のダメージが半減して再計算される（急所・すりぬけ等の例外を除く）

### Requirement: 定数ダメージの指定

システムは防御側の定数ダメージを指定できるようにし**なければならない（MUST）**。状態行（やけど/じゅうでん/
うちおとす/壁 の並び）に「定数」チップを置き、タップでダイアログを開く。ダイアログは**ステルスロック**
（登場時1回・いわ相性で算出）の切替と、**毎ターンのチップダメージ**（やどりぎ 1/8・砂嵐 1/16・どく 1/8・
やけど 1/16・のろい 1/4・ゴツゴツメット 1/6 等）の加算／クリアを提供する。表示ダメージ・割合にはチップ
（floor(HP×割合)）を加算し、確定数は旧 `ko_text` 準拠で entry（ステロ）を引いた実効 HP に対し毎ターン
constant×n を加味して算出する。

#### Scenario: 定数ダメージで確定数が変わる

- **WHEN** 「定数」チップからやどりぎ 1/8 を加算する
- **THEN** 毎ターンのチップ分だけ確定数（乱数 n 発）が早まり、表示ダメージ・割合にもチップが加算される

### Requirement: 技の威力・回数効果ボタン

システムは技の威力・回数の効果を切り替えられるようにし**なければならない（MUST）**。旧 champ-edge の
`waza.py`（multi_hit_values / add_power_values / power_hosei_values）を移植し、対象の技には技名の横に
効果ボタン（「×n」表示）を出す。連続技（タネマシンガン等）は回数、威力増加（アクロバット・じしん等）と
威力補正（はたきおとす等）は倍率を、タップで候補値を循環させる。選択値は `MoveState` の multiHit/
addPower/powerHosei としてエンジンへ渡し、各技は初期値（原典準拠）を既定で適用する。

#### Scenario: 連続技の回数を変える

- **WHEN** タネマシンガンの効果ボタン（×3）をタップする
- **THEN** 回数が次の候補（×4）に変わり、ダメージ範囲・割合・確定数が再計算される

#### Scenario: 威力倍率を変える

- **WHEN** アクロバットの効果ボタン（×2）をタップして ×1 にする
- **THEN** 威力倍率が変わり、ダメージが再計算される

### Requirement: 特性による天候・フィールドの自動反映

システムは場に出たポケモンの特性で天候・フィールドを自動反映し**なければならない（MUST）**。旧 champ-edge の
after_appear と同様に、ポケモンを選択（登場）した際、その特性が天候特性（すなおこし→砂嵐／ひでり・
ひひいろのこどう・メガソーラー→晴れ／あめふらし→雨／ゆきふらし→雪）またはフィールド特性（エレキメイカー・
ハドロンエンジン→エレキ／グラスメイカー→グラス／ミストメイカー→ミスト／サイコメイカー→サイコ）であれば、
中央列の天候/フィールドを対応する値に変更する。

#### Scenario: 天候特性で天候が変わる

- **WHEN** すなおこしのポケモンを選択する
- **THEN** 天候が「砂嵐」に変わり、ダメージが再計算される

### Requirement: 技ごとのダメージ可視化

システムは各技のダメージを可視化し**なければならない（MUST）**。各技についてダメージ範囲（min〜max）、
防御側 HP に対する割合（min–max%）、確定数を表示すること。確定数は旧 champ-edge の `ko_text` 準拠で、
n 発（1→4）で 16/16 倒せれば「確定 n 発」、それ未満は「乱数 n 発（k/16 の分数付き）」と表す。
技行は 1 行で「技名（左・固定幅、威力表示は省略）／ダメージ範囲（中央）／割合・確定数（右寄せ）」を並べ、
確定数の桁変化でも行がズレないこと。バーは旧 champ-edge
の HP バー準拠で**残り HP**を表し（明色＝確定で残る、暗色＝min〜max の不確定帯、灰＝確定で失う）、
色は最大ダメージ割合で 赤（≥80%）／黄（≥50%）／緑（<50%）とする。技は 5 枠（自分は空白可、相手は
5 枠目に暫定技）を横画面でスクロール無しに収める。

#### Scenario: ダメージ割合の可視化

- **WHEN** ある技を見る
- **THEN** その技のダメージ範囲・対 HP 割合・確定数が表示され、残り HP バーが割合に応じた色で示される

### Requirement: メニュー（隅のボタンから展開）

システムは中央列に操作アイコンとメニューを備え**なければならない（MUST）**。中央列は上から
メニュー（ハンバーガー）／HOME 使用率／類似パーティ検索／対戦記録のアイコン、天候・フィールドの
ドロップダウン、カウンター／タイマーのアイコンを縦に並べる（押しやすいよう間隔を空ける）。
ハンバーガーを押すとドロワーが開き、区切り線で「素早さ比較・重さ比較」「パーティ編集・ボックス編集」
「対戦履歴・対戦分析」「設定」の4グループを表示する。現状アイコン/項目の多くはプレースホルダ
（押下で「今後実装予定」）で、段階的に実装してよい。

#### Scenario: メニューを開く

- **WHEN** 中央列のハンバーガーメニューをタップする
- **THEN** 区切り線で分類された各メニュー項目（比較／パーティ／戦績／設定）が表示される

### Requirement: HOME 使用率情報の表示（相手スカウト）

システムは HOME（pokedb.tokyo 集計）の使用率情報を表示でき**なければならない（MUST）**。中央カラムの
ハンバーガーメニューの下に HOME アイコンを置き、タップで**相手（選択中）のポケモン**の使用率情報を
**画面いっぱいのポップアップ**で表示する（旧 champ-edge の HomeFrame＝player1 専用に相当）。ポップアップは
もちもの・とくせい・せいかく・どりょくち の4カテゴリを横並びにし、各カテゴリ内は値と使用率（%）を
使用率降順で縦に並べ、行タップでその値を相手ポケモンへ適用（努力値は 0-32 反映）する。**現在その
ポケモンに設定済みの値は塗りつぶしで強調**し、同じ値を再タップすると未選択（既定値：持ち物=なし／
性格=まじめ／努力値=0／特性=第一候補）に戻す。**わざ**の使用率は HOME ポップアップではなく
**技検索の候補先頭**に割合付きで提示する（タップで技確定）。データは旧アプリの `stats/home_*.csv`
（pokedb.tokyo のスナップショット）をアセット同梱して読む（実行時はローカル CSV を読むのみ。スクレイプ
更新は対象外）。メガなどフォーム名は基本形名へ正規化して引く。

#### Scenario: HOME 使用率の表示

- **WHEN** HOME アイコンをタップする
- **THEN** 相手の選択ポケモンの もちもの・とくせい・せいかく・どりょくち が使用率付きで縦並び表示される

#### Scenario: HOME 候補の適用

- **WHEN** HOME 情報の持ち物（または特性・性格・努力値）のエントリをタップする
- **THEN** その値が相手の選択ポケモンに設定され、ダメージが再計算される

#### Scenario: 技検索での HOME 上位提示

- **WHEN** 相手の技枠をタップして技検索を開く
- **THEN** そのポケモンの HOME 使用率上位の技が候補先頭に割合付きで表示され、タップで技が確定する
