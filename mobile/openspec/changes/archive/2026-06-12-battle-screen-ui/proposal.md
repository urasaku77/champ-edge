# 対戦画面（ダメージ計算）UI

## Why

Phase 1 で Dart ダメージエンジン（`decide-damage-engine-language`）が完成し、Flutter
プロジェクト（`decide-cross-platform-framework`）も整った。次は、ユーザーが実際に
ダメージ計算を行うための **対戦画面 UI** を確定させる必要がある。

現状はレイアウトと動作確認のための叩き台（プロトタイプ）が `lib/src/screens/home_screen.dart`
に存在するが、技の選び方・状態異常の扱い・HP 表示・性格選択などの**細かい仕様が未確定**で、
このまま作り込むと手戻りが発生する。旧 champ-edge（Tkinter デスクトップ版）の操作性を
踏襲しつつ、モバイル（横画面）向けに要件を明文化する。

## What Changes

- 旧アプリを踏襲した **左=自分 / 右=相手** の対戦レイアウトを正式仕様化する。
  - 各側: パーティ（6体・画像）＋選出、選択ポケモンの基本情報（画像・タイプ・テラス・
    特性・持ち物・性格・実数値）、技と相手選択ポケモンへのダメージ。
- 選択ポケモンの**編集方式**を確定する。
  - **技は文字入力で検索**して選ぶ（叩き台の固定プールではなく、技データから検索）。
  - 特性・持ち物は候補一覧から選ぶ。
  - **性格は表形式（5×5）**で選ぶ（叩き台のリストから変更）。
  - ステータス（努力値・個体値・レベル）を編集する。
- 計算に影響する**状態項目を UI に追加**する（エンジンは計算対応済み）。
  - 攻撃側: やけど等の状態異常、じゅうでん（ためわざ＝でんきタイプ威力2倍）、急所。
  - 場: 天候・フィールド、壁（リフレクター/ひかりのかべ/オーロラベール）。
  - テラスタイプの変更。
- **HP バー**を選択ポケモンに表示し、ダメージの視認性を高める。
- 全画面レイアウト（タイトル非表示）と、隅のメニューボタンから開く各種メニュー
  （天候・フィールド・タイマー・カウンター・対戦記録・パーティ編集・設定）。

> 備考: 技検索・特性/持ち物の正式な全候補は `pokemon.db`（`waza_data` 等）に依存するため、
> 本 change は **DB 連携（`decide-cross-platform-framework` の DB 組込みタスク）に依存**する。
> DB 連携が未完の間はサンプルデータで動作させ、連携後に実データへ差し替える。

## Capabilities

### New Capabilities
- `battle-screen-ui`: 対戦画面のレイアウト・選択ポケモン編集・状態/場の入力・HP 表示・
  ダメージ表示の UI 要件。

### Modified Capabilities
（なし）

## Impact

- 影響コード: `lib/src/screens/home_screen.dart`（叩き台を本仕様で再構築）、
  `lib/src/model/battle_pokemon.dart`、`lib/src/data/`（サンプル/オプションデータ、将来 DB 連携）。
- 依存: `service/damage_engine`（計算は実装済み・変更不要）、`pokemon.db`（技/特性/持ち物検索）。
- 既存の `decide-cross-platform-framework` の DB 組込み・横画面・権限タスクと連動する。
