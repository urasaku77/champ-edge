# 対戦記録の保存

## ADDED Requirements

### Requirement: 対戦記録の保存

システムは対戦結果を永続化できるようにし**なければならない（MUST）**（旧 champ-edge の
database/battle.py の battle テーブルの移植）。pokemon.db は読み取り専用アセットのため、対戦記録は
アプリのドキュメント領域に**書き込み可能な `battle.db`（sqflite）**を作成して保存する。記録1件は
原典と同一スキーマ（id・日時 date(epoch秒)・rule・result(1勝/2分/0負)・favorite・相手TN・相手レート・
メモ・自分パーティ番号/連番・自分ポケモン6・相手ポケモン6・自分選出4・相手選出4）を持つ。ポケモンは
pid 文字列、空き枠・未選出は `-1` とする。

#### Scenario: 対戦記録を保存する

- **WHEN** 中央列の「対戦記録」アイコンから登録ダイアログを開き、勝ち/分け/負けのいずれかを選ぶ
- **THEN** その時点の両パーティ・両選出・入力した相手TN/レート/メモ/お気に入りが battle.db に1件保存される

### Requirement: 対戦記録の登録ダイアログ

システムは対戦記録の登録ダイアログを備え**なければならない（MUST）**。ダイアログでは相手TN・相手レート・
メモ・お気に入りを**手動入力**でき（TN/レートの OCR 自動入力は将来フェーズ）、**匿名ボタン**で TN を
「トレーナー」に設定できる。勝ち/分け/負けのボタンで結果を指定して保存し、保存後はその旨を通知する。
ポケモンと選出は現在の盤面（自分/相手のパーティと選出）から取り込む。

#### Scenario: 匿名で記録する

- **WHEN** 登録ダイアログで「匿名」ボタンを押す
- **THEN** 相手TN欄に「トレーナー」が入る

### Requirement: 自分パーティの番号付け

システムは自分パーティに**番号・連番・タイトル**を付けて永続化できるようにし**なければならない（MUST）**。
パーティ編集（自分）に番号・連番・タイトルの入力欄を設け、入力内容は保存され、対戦記録の登録時に
番号・連番が記録に保存される（対戦分析の自分パーティ別絞り込みのキーになる）。

#### Scenario: 番号付きで記録される

- **WHEN** 自分パーティに番号「3」・連番「2」を設定して対戦を記録する
- **THEN** その記録の自分パーティ番号=3・連番=2 が保存され、後の分析で番号による絞り込みに使える
