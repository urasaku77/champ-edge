# スクレイピングデータのローカルキャッシュと依存機能

## ADDED Requirements

### Requirement: スクレイピングデータのローカルキャッシュ

システムはスクレイピング由来データをローカルキャッシュとして保持し、機能はそれを読むだけにし**なければならない（MUST）**。対象は全体使用率ランキング・シーズン定義・構築記事（旧 champ-edge の stats/ranking.txt・
recog/season.json・構築記事DB に相当）。現状はアプリ同梱の seed（`assets/data/scrape/{ranking,season,
kousei}.json`）を読み込む。データの**取得（更新）は機能本体と切り分け**、将来サーバー集約（API）または
バンドル更新で同じローカルキャッシュを差し替える（取得機構は本要件の対象外）。

#### Scenario: seed が無くても機能が壊れない

- **WHEN** スクレイピングデータが空または未整備のまま機能を使う
- **THEN** 例外を出さず、ランキング依存表示は無効化・シーズンは非表示・構築記事一致は0件として動作する

### Requirement: 使用率順ソートとトップN絞り（分析）

システムは対戦分析で**使用率順ソートとトップN絞り**を提供し**なければならない（MUST）**（旧 champ-edge の
使用率順・トップに絞る）。ソートは件数順／勝率順／**使用率順**（ランキング順）から選べ、**トップに絞る**は
OFF/10/30/50 を切り替えて、使用率ランキング上位 N に該当しない相手ポケモンを除外する。ランキングは
ローカルキャッシュ（ScrapeData.ranking）を用いる。

#### Scenario: 使用率順に並べトップ10に絞る

- **WHEN** 分析でソートを「使用率順」、トップを「10」にする
- **THEN** 相手ポケモンが使用率ランキング順に並び、上位10に入らないポケモンは一覧から除外される

### Requirement: シーズンによる期間選択（履歴・分析）

システムは履歴・分析で**シーズンを選んで期間を一括設定**できるようにし**なければならない（MUST）**
（旧 champ-edge のシーズン選択）。シーズンドロップダウンはローカルキャッシュ（ScrapeData.seasons）から
構成し、選択で from/to を設定する。シーズンデータが無ければドロップダウンは表示しない（期間プリセットで代替）。

#### Scenario: シーズンで期間を設定する

- **WHEN** シーズンドロップダウンから1つ選ぶ
- **THEN** そのシーズンの開始日〜終了日が期間に設定され、一覧/集計が更新される

### Requirement: 構築記事ベース検索の実データ

システムは類似パーティ検索の構築記事照合にローカルキャッシュの構築記事（ScrapeData.kousei）を用い**なければならない（MUST）**。seed（および手動登録の DB 分）の構築記事パーティと相手パーティを照合し、
並び一致／中身一致を区別して表示する。

#### Scenario: 構築記事 seed と一致する

- **WHEN** 相手パーティが構築記事 seed のいずれかと構成一致する状態で類似パーティ検索を開く
- **THEN** その構築記事が一致結果（並び一致/中身一致）として表示され、リンクを開ける
