# タイマー/カウンタの直接表示・詳細ジェスチャのダブルタップ統一

## MODIFIED Requirements

### Requirement: タイマー・カウンタ

システムは対戦用のタイマーとカウンタを備え**なければならない（MUST）**（旧 champ-edge の
TimerFrame / CountersFrame 相当）。タイマー・カウンタは**ポップアップではなく中央列に直接表示**し、
表示はサイズ固定でレイアウトがずれないようにする（文字は小さくてよい）。中央列の幅は従来どおりで
ポケモン情報の幅を変えない。カウンタ/タイマーは押しやすいよう間隔をあける（スクロール・縦オーバー
フローしない範囲で）。

- **タイマー**：中央列に残り時間（mm:ss）を常時表示する。20 分（1200 秒）からのカウントダウンで、
  **一回押し＝スタート/ストップ・長押し＝リセット**（ボタンは設けない）。表示の背景色は原典と同一の
  閾値で変化する（残り 10 分以上＝緑／10 分未満＝青／5 分未満＝黄／3 分未満＝赤）。
- **カウンタ**：中央列にカウンタ[0] を直接表示する。`− 値 ＋`（0〜99）で、− で減算・＋ で加算・
  数字の長押しで 0 リセット。さらに**メニューにカウンタ項目**があり、全体ポップアップで**3 個**の
  カウンタ（各タイトル入力・−/＋/0・全リセット）を扱える。カウンタ[0] は中央列と共有。値とタイトルは
  **対戦記録または明示リセットまで永続化**する（PP 管理等に利用）。

#### Scenario: タイマーの開始と色変化

- **WHEN** 中央列のタイマーを一回押してスタートし、残り時間が 3 分未満になる
- **THEN** カウントダウンが進行し、表示の背景が赤になる（長押しでリセットできる）

#### Scenario: カウンタで PP を管理する

- **WHEN** メニューのカウンターを開きタイトルを入れて「＋」を3回押し、アプリを再起動する
- **THEN** カウントとタイトルが保持されている（中央列のカウンタ[0] にも反映される）

### Requirement: 特性・持ち物の編集

選択ポケモンの特性・持ち物をタップで変更できなければならない（MUST）。特性はそのポケモンの
候補（DB の ability1/2/3、未接続時はサンプル）から、持ち物は**文字入力検索**（DB の item_data）で
選ぶ。item_data は表記が混在（カタカナ/ひらがな）のため、検索は**生入力・カタカナ・ひらがなのいずれかで
部分一致**させる（いのちのたま／こだわりスカーフ等いずれも引ける）。**特性・持ち物の選択時は、その相手
ポケモンの HOME 使用率を各候補の横に表示し、使用率の高い順に並べ替える**。変更するとダメージが
再計算されること。チップは選択値のみ表示し、持ち物が「なし」のときは「もちものなし」と表示する。
さらに、**特性・持ち物のチップをダブルタップすると、その効果詳細（`ability_data.effect` /
`item_data.effect`）をダイアログ表示する**（旧 champ-edge の右クリック効果詳細に相当）。

**特性値の切替**：旧 champ-edge の `ABILITY_VALUES` を移植し、条件付き特性は値を切り替えられること。
有効/無効型（しんりょく・もうか・げきりゅう・スナイパー・むしのしらせ・ねつぼうそう・どくぼうそう・
プラス・マイナス＝デフォルト無効／へんげんじざい・リベロ・マルチスケイル・ファントムガード・
ふしぎなうろこ・アナライズ・かたやぶり・スロースタート・よわき・テラスシェル＝デフォルト有効／
ふかしのこぶし・かんつうドリル＝デフォルト無効・有効でダメージ×1/4）と、多値型（そうだいしょう
1.0/1.1/1.2・とうそうしん 1.0/1.25/0.75・こだいかっせい/クォークチャージ なし/A/B/C/D/S）がある。

- 値は特性チップ**ダブルタップ**の効果詳細ダイアログ内のチップで切り替え、即座にダメージへ反映する。
- **特性を変更すると、その特性のデフォルト値（テーブル先頭）へ自動リセット**する
  （原典 `set_default_ability_value` と同一）。値はパーティに永続化する。
- 値を持つ特性はチップに現在値を `[値]` で併記する。
- 値はエンジンの `abilityValue` へ渡り、条件付き特性補正（マルチスケイル半減・こだいかっせい等）が
  実際に計算へ反映されること。

#### Scenario: 特性の変更

- **WHEN** 特性をタップして候補から別の特性を選ぶ
- **THEN** 選択ポケモンの特性が更新され（特性値はデフォルトへリセット）、ダメージが再計算される

#### Scenario: ふかしのこぶしを有効にする

- **WHEN** ふかしのこぶし（デフォルト無効）の特性チップをダブルタップし、値を「有効」に切り替える
- **THEN** プロテクト貫通を想定し、その攻撃のダメージが通常の 1/4 で表示される

#### Scenario: マルチスケイルが防御計算に乗る

- **WHEN** 防御側がマルチスケイル（デフォルト有効）を持つ
- **THEN** 被ダメージが半減して計算される（値を「無効」にすると通常に戻る）

#### Scenario: 持ち物の検索（混在表記）

- **WHEN** 持ち物検索で「いのち」と入力する
- **THEN** ひらがな表記の「いのちのたま」が候補に表示される

#### Scenario: 効果詳細の表示

- **WHEN** 特性チップ（または持ち物チップ）をダブルタップする
- **THEN** その特性/持ち物の効果説明がダイアログで表示される

### Requirement: 技ごとのダメージ可視化

システムは各技のダメージを可視化し**なければならない（MUST）**。各技についてダメージ範囲（min〜max）、
防御側 HP に対する割合（min–max%）、確定数を表示すること。確定数は旧 champ-edge の `ko_text` 準拠で、
n 発（1→4）で 16/16 倒せれば「確定 n 発」、それ未満は「乱数 n 発（k/16 の分数付き）」と表す。
技行は 1 行で「技名（左・固定幅、威力表示は省略）／ダメージ範囲（中央）／割合・確定数（右寄せ）」を並べ、
確定数の桁変化でも行がズレないこと。バーは旧 champ-edge
の HP バー準拠で**残り HP**を表し（明色＝確定で残る、暗色＝min〜max の不確定帯、灰＝確定で失う）、
色は最大ダメージ割合で 赤（≥80%）／黄（≥50%）／緑（<50%）とする。技は 5 枠（自分は空白可、相手は
5 枠目に暫定技）を横画面でスクロール無しに収める。
**操作の統一**：技行は**タップで技変更・長押しで加算ツール・ダブルタップで技詳細**（行全体で統一。
領域分割はしない）。

#### Scenario: ダメージ割合の可視化

- **WHEN** ある技を見る
- **THEN** その技のダメージ範囲・対 HP 割合・確定数が表示され、残り HP バーが割合に応じた色で示される

#### Scenario: 技行の操作

- **WHEN** 技行を長押しする／ダブルタップする
- **THEN** 加算ツール／技詳細ダイアログがそれぞれ開く（タップは技変更）

### Requirement: 加算ツール

システムは複数技の合算ダメージを確認できるようにし**なければならない（MUST）**。技行**全体を長押し**
すると加算ツールのポップアップを開く（旧 champ-edge の `MultiWazaDamageWindow`＝ダメージ表示
右クリック相当）：

- 攻撃側の設定済み技がボタンで並び、押すたびにその技のダメージが累積リストへ追加される
  （同じ技を複数回押すと複数回分加算）。
- 回復ボタン（オボンのみ＝最大HP×1/4・たべのこし＝×1/16・混乱実＝×1/3・やどりぎのタネ＝×1/8）を
  押すと回復量が減算行として追加される。
- 各行は 技名/回復名・ダメージ・割合（技は単発の確定数付き）を表示し、**行をタップするとその行だけ
  削除**して再計算される。「クリア」ボタンで全行リセット。
- 合計欄にネットダメージ実数（min−max）・割合・**KO 判定**を表示する。KO 判定は各技の 16 値ダメージ
  分布（定数ダメージ加算済み）の直積を DP で集計し、しきい値＝実効HP（ステルスロック分を差し引き、
  最低1）＋回復合計 とし、全組み合わせで KO なら「確定KO」、一部なら「乱数KO (確率%)」とする。
- やけど・急所・定数ダメージ等の設定は各技のダメージに反映済みの値を用いる。

#### Scenario: 複数技の合算と KO 判定

- **WHEN** 加算ツールで技Aを2回押し、技Bを1回押す
- **THEN** 3行が追加され、合計欄に合算ダメージ・割合と 確定KO/乱数KO(確率%) が表示される

#### Scenario: 回復の減算と行削除

- **WHEN** たべのこしボタンを押し、その後いずれかの行をタップする
- **THEN** 回復分が合計から減算され、タップした行だけが削除されて再計算される

### Requirement: 技詳細の表示

システムは技の詳細を確認できるようにし**なければならない（MUST）**。技行を**ダブルタップ**すると、
`waza_data` の タイプ・分類・威力・命中・PP・接触/非接触・まもる可否・対象 と説明文（description）を
ダイアログで表示する（旧 champ-edge の技右クリック相当）。特性・持ち物のダブルタップ効果詳細と同じ流儀。

#### Scenario: 技詳細を見る

- **WHEN** 設定済みの技行をダブルタップする
- **THEN** その技のタイプ/分類/威力/命中/PP と説明文がダイアログ表示される

### Requirement: 定数ダメージの指定

システムは防御側の定数ダメージを指定できるようにし**なければならない（MUST）**。状態行（やけど/じゅうでん/
うちおとす/壁 の並び）に「定数」チップを置き、タップでダイアログを開く。ダイアログは**ステルスロック**
（登場時1回・いわ相性で算出）の切替と、**毎ターンのチップダメージ**（やどりぎ 1/8・砂嵐 1/16・どく 1/8・
やけど 1/16・のろい 1/4・ゴツゴツメット 1/6 等）の加算／クリアを提供する。表示ダメージ・割合にはチップ
（floor(HP×割合)）を加算し、確定数は旧 `ko_text` 準拠で entry（ステロ）を引いた実効 HP に対し毎ターン
constant×n を加味して算出する。**定数が設定されているときはチップに値のみ表示**し（「定数」ラベルは省く）、
未設定時のみ「定数なし」と表示する。

#### Scenario: 定数ダメージで確定数が変わる

- **WHEN** 「定数」チップからやどりぎ 1/8 を加算する
- **THEN** 毎ターンのチップ分だけ確定数（乱数 n 発）が早まり、表示ダメージ・割合にもチップが加算される

#### Scenario: 定数チップのラベル省略

- **WHEN** 定数（ステロやどく等）を設定する
- **THEN** チップは値のみ（例「ステロ+1/10」）を表示し、未設定時は「定数なし」と表示する

### Requirement: メニュー（隅のボタンから展開）

システムは中央列に操作アイコンとメニューを備え**なければならない（MUST）**。中央列は上から
メニュー（ハンバーガー）／HOME 使用率／類似パーティ検索／対戦記録のアイコン、天候・フィールドの
ドロップダウン、カウンタ／タイマーの直接表示を縦に並べる（押しやすいよう間隔を空ける）。
ハンバーガーを押すとドロワーが開き、区切り線で「素早さ比較・重さ比較・カウンター」「パーティ編集・
ボックス編集」「対戦履歴・対戦分析」「設定」の4グループを表示する。**素早さ比較・重さ比較・カウンター・
タイマーは実装済み**（各要件参照。カウンターはメニューから3個のポップアップを開く）。残りの項目は
プレースホルダ（押下で「今後実装予定」）で、段階的に実装してよい。

#### Scenario: メニューを開く

- **WHEN** 中央列のハンバーガーメニューをタップする
- **THEN** 区切り線で分類された各メニュー項目（比較／パーティ／戦績／設定）が表示される

#### Scenario: メニューからカウンターを開く

- **WHEN** メニューの「カウンター」をタップする
- **THEN** 3個のカウンタを扱う全体ポップアップが開く
